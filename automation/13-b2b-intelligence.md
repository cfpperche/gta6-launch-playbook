# Automation brief — 13. B2B GTA 6 intelligence product

> **Opportunity:** a subscription intelligence product on the GTA 6 economy, sold
> B2B to studios, funds, brands, and agencies. Catalogue:
> [`../opportunities/README.md`](../opportunities/README.md).
> **This brief:** how far AI takes development + operation toward 100%.
> **Tier 3** — a blind-spot addition; see [`README.md`](README.md).

## Verdict — ~85%

Data collection, aggregation, analysis, and report-writing are exactly the
autonomous-research pattern that shipped in 2026 — collector agents, analysis
agents, report-writer agents with fact-check. The pipeline automates cleanly.
What does not: B2B sales relationships, the credibility to be *trusted by
institutions*, knowing which questions institutions actually pay to answer, and
accountability for data accuracy (institutions churn — or sue — over bad data).

## Build phase — automating development

- An agent fleet builds the data pipeline — collectors, store, dashboards, API —
  via agentic coding.
- Collector agents are wired to public sources: Twitch/YouTube APIs, store and
  pre-order data, on-chain data, social sentiment, RP-server metrics.
- **Start now:** data history compounds — the dataset is itself the moat, and it
  only accrues if collection starts early.

## Operate phase — automating execution

- Collector agents run 24/7; an analysis agent turns raw data into insight; a
  report-writer + fact-check agent drafts subscriber reports and the data feed.
- A distribution agent handles the newsletter/portal and AEO.
- A billing agent runs B2B subscriptions and renewals.

## The AI stack (2026)

Scraping/collector agents · agent-built data infrastructure · analysis agents
(Claude) · report-writer + fact-check agents · dashboard · agentic coding
(Claude Code) for the whole platform.

## Human-in-the-loop — the supervisor role

- **B2B sales** — institutional subscriptions close human-to-human.
- **Credibility & authority** — institutions buy intelligence from a name they
  trust; that reputation is built by a person.
- **Defining the questions** — knowing which analyses funds and studios actually
  pay for is market judgment, not a data task.
- **Data-accuracy accountability** — a wrong number sold to an institution is a
  liability; a human owns the quality bar.

## What blocks the last mile to 100%

Institutional trust is personal. B2B sales is relationship work. Accuracy
accountability is legal, not technical.

## Failure modes of the automated approach

Automate the data and still fail: a beautiful dashboard answering questions no
institution pays for. Data-quality errors at scale destroy institutional trust
instantly and irreversibly. Free retail-grade alternatives can erode the
perceived value of a paid feed if the differentiation is not sharp.
