# Automation briefs

One brief per opportunity (`01`–`17`), each answering a single question:

> **How far can AI take the development *and* the day-to-day operation of this
> opportunity toward 100% — with humans allowed in the loop?**

This folder exists because of a goal set on 2026-05-20: *before* scoring the
opportunities against the owner's skills, first map how much of each could run on
an AI agent fleet. That reframes the skill question itself — see below.

## Why this changes the evaluation

The [`../evaluation/`](../evaluation/) scorecard has a `skill fit` column. If an
opportunity can be ~90% agent-operated, the skill that matters is no longer the
domain craft (writing, editing, Lua, trading) — it becomes **agent
orchestration**: designing, supervising, and improving a fleet of AI agents.
These briefs let the owner answer the skill question in 2026 terms, not 2013 terms.

## What "100% automation" realistically means in 2026

Nothing here hits literal 100%. The honest target — and the shape the 2026
industry has converged on — is **one human supervising an agent fleet.**
ServiceNow, SAP and others now ship "autonomous workforce" suites on exactly that
frame: the operator becomes a supervisor of agents. So "100%" reads as: *a solo
operator orchestrating agents and doing none of the production work themselves.*

Four things stay human across every opportunity:

1. **Deciding what to do** — taste, market sense, the call on what is worth making.
2. **Accountability** — legal, platform ToS, tax, the name on the account.
3. **Trust & relationships** — community leadership, B2B deals, the reason anyone
   pays a person rather than a feed.
4. **The genuinely novel edge case** — the situation no agent has a playbook for.

## The Tier 1 / Tier 2 split is sharper here

- **Tier 1 (build a business):** automation is a genuine force-multiplier — it
  lets one person run what used to need a team. Automating it is good.
- **Tier 2 (allocate capital):** automation amplifies the *loss* profile exactly
  as efficiently as the gain. A bot flawlessly executing a 95%-rug memecoin
  strategy just reaches ruin faster and at scale. For Tier 2, a high
  "automatable %" is a **warning, not a feature** — every Tier 2 brief says so.

## Brief template

Each brief follows the same shape: **Verdict · Build-phase automation ·
Operate-phase automation · AI stack (2026) · Human-in-the-loop · What blocks the
last mile · Failure modes of the automated approach.**
