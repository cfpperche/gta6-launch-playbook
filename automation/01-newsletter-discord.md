# Automation brief — 1. Newsletter / Discord "alpha"

> **Opportunity:** a paid newsletter or Discord that becomes the information hub
> for GTA 6. Catalogue: [`../opportunities/README.md`](../opportunities/README.md).
> **This brief:** how far AI takes development + operation toward 100%.

## Verdict — ~85%

The production pipeline (research → write → fact-check → publish) is a *solved*
2026 agent workflow, and Discord moderation is ~70%+ automatable. What cannot be
automated is the **reason anyone pays**: a genuine information edge — "alpha". An
agent can aggregate every public leak; it cannot *have a source*. So the pipeline
is ~90% automatable, but the product is only as valuable as the human-supplied
edge feeding it. Automate the production; the human owns the edge.

## Build phase — automating development

- An agent fleet scaffolds the whole stack: newsletter platform (Beehiiv), Discord
  server, payment/membership rails. Agentic coding builds any custom tooling.
- A **research agent** continuously monitors leak sources, the Rockstar newswire,
  Take-Two filings, RP-server news, and streamer activity.
- A **writer agent** drafts each issue from that feed; a **fact-check agent**
  verifies every claim against sources before publish. This exact pipeline —
  research → write → fact-check → push to Beehiiv — ships in 2026.

## Operate phase — automating execution

- Issues auto-assembled and published on a fixed cadence.
- Discord: a GPT-4o-class bot handles `/ask`, `/summarize`, onboarding, and
  auto-moderation (~70% of repetitive mod load), plus raid prediction.
- A growth agent runs cross-posting, SEO, and referral mechanics.
- A monetization agent manages the free → paid funnel, dunning, and churn winback.
  Sponsorship *outreach* can be agent-drafted; *closing* is human.

## The AI stack (2026)

Web-monitoring research agents · Claude/GPT writer + fact-check agents · Beehiiv
automation · Discord AI bot (MEE6-class or custom GPT-4o bot) · Claude Code for
custom tooling · analytics agent.

## Human-in-the-loop — the supervisor role

- **Supply the edge.** The owner's real access/insight is the input the whole
  pipeline refines. Without it the newsletter is a commodity aggregator.
- **Editorial voice & approval.** Post the March-2026 penalty wave, undifferentiated
  AI content is distrusted and demoted — a human voice and final pass are required.
- **Close sponsorships; own the brand and the legal entity.**
- **Handle community crises** an agent would mishandle.

## What blocks the last mile to 100%

The "alpha" is by definition non-public — an agent has no sources. Trust accrues
to a person/brand. Sponsors buy from people. These are not engineering gaps.

## Failure modes of the automated approach

A fully-automated newsletter converges on the same public-leak aggregation that
everyone else's agent produces → zero edge → no paid conversion. Automation makes
*production* free, which means production stops being the moat — the moat moves
entirely onto the human edge. An over-automated, bot-only Discord feels dead and
churns members.
