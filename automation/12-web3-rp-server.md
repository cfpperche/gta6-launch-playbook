# Automation brief — 12. Web3 RP server

> **Opportunity:** a GTA 6 roleplay server monetized with Web3 — on-chain
> reputation, NFT car titles, crypto subscription payments.
> Catalogue: [`../opportunities/README.md`](../opportunities/README.md).
> **This brief:** how far AI takes development + operation toward 100%.

## Verdict — ~75% technically automatable, blocked by non-technical walls

This is opportunity [#7](07-rp-server.md) plus a Web3 layer. The #7 automation
stack applies in full, and the Web3 additions — smart contracts for NFT titles,
on-chain reputation, crypto payment rails — are themselves agentic-codable. But
the binding constraints here are **legal and contractual**: Rockstar's terms of
service most likely prohibit crypto integration outright, and regulatory
uncertainty around NFTs and crypto payments is real. **No amount of automation
moves a ToS clause.**

## Build phase — automating development

- Everything in [#7](07-rp-server.md)'s build phase (world-building, server code,
  AI NPCs), plus:
- **Smart contracts** (NFT vehicle titles, on-chain reputation, subscription
  payments) — agent-generated and agent-audited, with a hard caveat: contract
  security needs human review. An exploited contract holding real value is
  unrecoverable.

## Operate phase — automating execution

- [#7](07-rp-server.md)'s operate phase, plus an on-chain agent managing mint,
  transfer, and payment flows.

## The AI stack (2026)

[#7](07-rp-server.md)'s stack + smart-contract coding agents + contract-audit
agents + on-chain operations agents.

## Human-in-the-loop — the supervisor role

- **Legal and regulatory navigation** — the dominant task here, and entirely
  human: a real reading of GTA 6's ToS, securities/NFT regulatory exposure, and
  jurisdiction.
- **Smart-contract security sign-off** — an automated audit is not sufficient for
  code that holds real value.
- Everything in [#7](07-rp-server.md)'s human core — community leadership,
  capital, vision.

## What blocks the last mile to 100%

The blockers are non-technical: ToS, regulation, legal accountability. Automation
has nothing to offer against a contract clause or a regulator.

## Failure modes of the automated approach

Build the entire Web3 server, then GTA 6 ToS bans crypto integration → dead, with
the full sunk cost. A smart-contract exploit drains real funds irreversibly.
Regulatory action against NFT/crypto monetization. This opportunity combines the
highest technical barrier on the list with the highest non-technical risk.
